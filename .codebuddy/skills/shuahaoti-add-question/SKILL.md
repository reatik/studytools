# 刷好题题库题目添加技能

## 目标文件
`e:\py\pyfile\github-studytools\subjects\english\shuahaoti\tiku.json`

## 使用方式
用户提供题目时，会给出：
- 题目题干（question）
- 题目类型（type）：选择 或 填空
- 年级（grade）
- 知识点（knowledge_point）
- 如果是选择题，还会提供选项（options）

AI 需要自动补全以下字段后，将新题目追加到 JSON 数组中。

---

## JSON 题目结构

### 选择题 (type: "选择")
```json
{
    "id": "<自增编号>",
    "type": "选择",
    "grade": "<年级>",
    "knowledge_point": "<知识点>",
    "question": "<题干>",
    "options": [
        "A. <选项A>",
        "B. <选项B>",
        "C. <选项C>",
        "D. <选项D>"
    ],
    "answer": "<正确选项字母>. <正确选项内容>",
    "meaning": "<中文翻译>",
    "explanation": "<中文解析>"
}
```

### 填空题 (type: "填空")
```json
{
    "id": "<自增编号>",
    "type": "填空",
    "grade": "<年级>",
    "knowledge_point": "<知识点>",
    "question": "<题干，用 ______ 或 __ 表示填空位置>",
    "options": [],
    "answer": "<正确答案>",
    "meaning": "<中文翻译>",
    "explanation": "<中文解析>"
}
```

### 可选字段
- `"unit"`: 部分年级的题目可能包含单元信息，如 `"unit": "U5"`

---

## 字段填写规范

### 1. id
- 读取 `tiku.json` 中 `questions` 数组最后一个题目的 `id`
- 如果最后 id 是纯数字（如 "105"），新题目 `id = 最大数字 id + 1`（转为字符串）
- 如果最后 id 是字母+数字格式（如 "C430"），提取数字部分 +1，如 "C431"

### 2. grade（年级）
可用年级值包括但不限于：
- `"中考难题"` - 中考难度题目
- `"六年级下"`, `"六年级上"` - 小学六年级
- `"五年级"`, `"四年级"`, `"三年级"` - 小学其他年级
- `"小升初"` - 小升初衔接
- `"七年级"`, `"八年级"` 等初中年级

### 3. knowledge_point（知识点）
可用知识点包括但不限于：
- `"名词"`, `"代词"`, `"冠词"`, `"数词"`
- `"介词"`, `"形容词"`, `"副词"`, `"动词"`
- `"连词"`, `"时态"`, `"语态"`, `"从句"`
- `"句型转换"`, `"比较级"`, `"最高级"`
- **词性转换题**：填写转换后的词性（即答案的词性），而不是写"词性转换"。例如 encourage → encouragement，答案 encouragement 是名词，写 `"名词"`

### 4. type（题目类型）
- `"选择"`：题干有缺口，提供 A/B/C/D 四个选项
- `"填空"`：题干有 `______` 或 `__` 表示填空位置，需要填写词语或短语
- `"句型转换"`：同义句转换，题干有两处及以上 `______` 填空，需要填入转换后的词语。**如涉及「对画线部分提问」等题型，原句划线处和填空位都要用 `______` 表示，不能遗漏。**

### 5. question（题干）
- 英文题干，保持与现有题目一致的风格
- 选择题目中如有缺口用 `______` 或 `__` 表示
- 填空题目中用 `______` 表示填空位置，提示词用 `(word)` 括起来
- **句型转换题（含划线部分）**：图片中划线部分的内容用 `<u>...</u>` 包裹表示，填空位用 `______`，两者缺一不可
- **题干中如需换行，使用 `<br>`，不要用 `\n`**

### 6. options（选项）
- 选择题：数组，每个元素为字符串格式 `"A. xxx"`, `"B. xxx"` 等
- 填空题：空数组 `[]`

### 7. answer（答案）
- **选择类型**：格式为 `"<选项字母>. <选项内容>"`，如 `"D. exercise, exercises"`、`"C. a, /"`
  - 注意字母后是英文句点 `.`，不是中文句号
- **填空类型**：直接写答案，如 `"children's"`、`"has"`、`"to keep"`
- **句型转换类型**：多空答案用分号 `;` 隔开，如 `"uses;again"`

### 8. meaning（中文翻译）
- **必须用中文编写**，翻译题目的完整句意
- 对于选择题目：翻译正确选项对应的完整句子含义
- 对于填空题目：翻译填入正确答案后的完整句子含义
- **如需换行，使用 `<br>`，不要用 `\n`**
- 示例：`"汤姆，你应该多做锻炼。不要总是坐在书桌前忙着做你的练习题。"`
- 放在 `answer` 之后、`explanation` 之前

### 9. explanation（解析）
- **必须用中文编写**，简单易懂，说清关键考点即可
- **言简意赅，控制在 1-2 句**，不啰嗦
- **不要翻译句子，不要写「句意」**（翻译已在 meaning 字段中），直接说明考点
- **不要额外加入易错段落**，重点说清考点即可。如确实要提醒，一句话带过
- 示例：
  - 好：`"固定搭配 do one's share 意为「尽一份力」。"`
  - 好：`"考查动词不定式作目的状语。"`
  - 好：`"classical 指古典的，classic 指经典的，此处古典戏剧用 classical。"`

### 10. unit（可选，单元信息）
- 如果用户指定了单元（如 U5），添加 `"unit"` 字段
- 放在 `grade` 之后

---

## 执行步骤

1. **先读取文件末尾**，获取最后一个题目的 id
2. **分析用户提供的题目信息**：确定 type、grade、knowledge_point
3. **如果是选择题**：生成 options 数组和 answer
4. **如果是填空题**：确定 answer 文本，options 设为空数组
5. **编写 meaning**（题目的中文翻译）
6. **编写 explanation**（中文语法解析）
7. **生成新 id**（自增编号）
8. **使用 replace_in_file 工具**，在 JSON 数组的 `questions` 数组末尾 `]` 之前追加新题目
9. **追加格式**：在最后一个 `}` 后面加逗号，然后插入新题目 JSON

## 追加示例

假设文件末尾是：
```
        {
          "id": "105",
          "type": "填空",
          ...
        }
  ]
}
```

追加后变为：
```
        {
          "id": "105",
          "type": "填空",
          ...
        },
        {
          "id": "106",
          "type": "选择",
          "grade": "中考难题",
          "knowledge_point": "名词",
          "question": "...",
          "options": [
            "A. ...",
            "B. ...",
            "C. ...",
            "D. ..."
          ],
          "answer": "...",
          "meaning": "...",
          "explanation": "..."
        }
  ]
}
```

注意：
- 保持与已有题目一致的缩进风格（6空格或4空格，以现有文件为准）
- 对象最后一个字段后面不加逗号
- 确保 JSON 格式正确，所有字符串用双引号
- 追加前检查前一个题目是否有逗号后缀，如果没有则先添加逗号
